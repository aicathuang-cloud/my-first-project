## 功能介绍

Task Status Indicator 为 OpenClaw 添加实时任务状态可视化功能，让你清楚知道 AI 正在做什么。

### 核心功能

| 功能 | 说明 |
|------|------|
| ⏱️ **实时计时** | 显示当前任务已运行时间 |
| 🔗 **工具链追踪** | 显示嵌套工具调用关系 |
| 📊 **进度可视化** | ASCII 进度条，支持脉冲动画 |
| ✅ **完成标记** | 任务成功/失败清晰反馈 |
| 🎯 **零侵入** | 不修改 OpenClaw 源码 |

### 效果预览

```
# 运行中
[🦐 腾云虾] 正在执行: browser.snapshot | 已耗时 12s | 工具链: browser → eval
[████████░░] 步骤 3 | 最后活动: 2s 前

# 完成
✅ [🦐 腾云虾] browser.snapshot | 完成 | 耗时 12s | 步骤 3

# 失败
❌ [🦐 腾云虾] browser.snapshot | 失败 | 耗时 45s | Timeout
```

## 安装步骤

### 方式一：一键安装（推荐）

```bash
# 1. 解压安装包
tar -xzf task-status-indicator-v0.1.0.tar.gz
cd task-status-indicator

# 2. 运行安装脚本
./install.sh
```

安装脚本会自动：
- 添加执行权限
- 检测你的 Shell 类型
- 可选：添加 `ocs` 别名

### 方式二：手动安装

```bash
# 1. 解压
tar -xzf task-status-indicator-v0.1.0.tar.gz

# 2. 添加权限
chmod +x task-status-indicator/*.js task-status-indicator/*.sh

# 3. 添加到 PATH（可选）
echo 'export PATH="$PATH:/path/to/task-status-indicator"' >> ~/.bashrc
```

### 验证安装

```bash
# 运行测试
node task-status-indicator/test.js

# 预期输出：

# ✓ formatTime - 毫秒
# ✓ formatTime - 秒
```

## 配置说明

### 默认配置

编辑 `status-renderer.js` 头部：

```javascript
const defaultConfig = {
  emoji: '🦐',                    // 状态条图标
  agentName: '腾云虾',             // 显示名称
  refreshIntervalMs: 500,         // 刷新间隔（毫秒）
  pulseThresholdMs: 3000,         // 脉冲模式阈值（毫秒）
  maxToolChainLength: 3,          // 工具链最大显示长度
  showToolChain: true,            // 是否显示工具链
  barLength: 10                   // 进度条长度
}
```

### 配置项说明

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `emoji` | 🦐 | 状态条左侧图标 |
| `agentName` | 腾云虾 | AI 助手显示名称 |
| `refreshIntervalMs` | 500 | 状态刷新频率，越小越实时但耗电 |
| `pulseThresholdMs` | 3000 | 单步骤超过此时长，进度条变脉冲动画 |
| `maxToolChainLength` | 3 | 工具链超过此长度显示省略号 |
| `showToolChain` | true | 是否显示工具调用链 |
| `barLength` | 10 | 进度条字符长度 |

### 自定义示例

```javascript
// 改为机器人图标和英文
const config = {
  emoji: '🤖',
  agentName: 'Assistant',
  refreshIntervalMs: 300  // 更快刷新
}

const wrapper = new OpenClawWrapper(config);
```

## 状态条解读

### 运行中状态

```
[🦐 腾云虾] 正在执行: browser.snapshot | 已耗时 12s | 工具链: browser → eval
[████████░░] 步骤 3 | 最后活动: 2s 前
```

| 部分 | 含义 |
|------|------|
| `[🦐 腾云虾]` | AI 标识 |
| `browser.snapshot` | 当前执行的工具 |
| `已耗时 12s` | 任务运行时间 |
| `工具链: browser → eval` | 工具调用嵌套关系 |
| `[████████░░]` | 进度条 |
| `步骤 3` | 当前步骤数 |
| `最后活动: 2s 前` | 距离上次工具调用的时间 |

### 脉冲动画模式

当单步骤运行超过 `pulseThresholdMs`（默认 3 秒）：

```
[🦐 腾云虾] 正在执行: read | 已耗时 30s
[░░░▓░░░░░] 步骤 1 | 最后活动: 30s 前
         ↑
    小方块左右滑动，表示"仍在运行，没卡死"
```

### 完成状态

```
✅ [🦐 腾云虾] browser.snapshot | 完成 | 耗时 12s | 步骤 3
```

- `✅`：成功标记
- 定格显示，保留在历史输出中
- 显示总耗时和总步骤数

### 失败状态

```
❌ [🦐 腾云虾] browser.snapshot | 失败 | 耗时 45s | Timeout
```

- `❌`：失败标记
- 显示错误类型（Timeout / Error / 等）
- 保留在历史中便于排查

## 故障排除

### 问题 1：状态条不显示

**现象**：启动后只有普通 OpenClaw 输出，没有状态条。

**排查步骤**：

1. 检查 Node.js 版本
   ```bash
   node --version  # 需要 ≥ 18.0
   ```

2. 检查文件权限
   ```bash
   ls -la openclaw-status-wrap.js  # 应该有 x 权限
   chmod +x openclaw-status-wrap.js
   ```

3. 检查 OpenClaw 是否可用
   ```bash
   which openclaw  # 确认在 PATH 中
   ```

### 问题 2：状态条位置错乱

**现象**：状态条和输出内容混在一起。

**原因**：终端不支持 ANSI 控制序列。

**解决**：
- 使用现代终端（iTerm2、Windows Terminal、VS Code 终端）
- 检查 `TERM` 环境变量：`echo $TERM` 应该包含 `xterm` 或 `screen`

### 问题 3：状态条闪烁

**现象**：状态条闪烁或刷新不平滑。

**解决**：
- 增大刷新间隔（改 `refreshIntervalMs` 为 1000）
- 减少终端窗口大小（避免重排）

### 问题 4：工具名解析错误

**现象**：状态条显示的工具名不正确。

**解决**：
- 编辑 `openclaw-status-wrap.js`
- 在 `parseLine` 方法中添加新的匹配模式

### 问题 5：别名不生效

**现象**：输入 `ocs` 提示 command not found。

**解决**：
```bash
# 重新加载 shell 配置
source ~/.bashrc  # 或 ~/.zshrc

# 或直接使用完整路径
node /full/path/to/openclaw-status-wrap.js
```

## 常见问题

### Q1: 这个工具会影响 OpenClaw 的性能吗？

**A**: 不会。状态条渲染在独立进程中运行，占用资源极少（< 50MB 内存，< 5% CPU）。

### Q2: 可以和其他 OpenClaw 插件一起用吗？

**A**: 可以。本工具通过包装器模式工作，不修改 OpenClaw 内部，与其他插件无冲突。

### Q3: 支持 Windows 吗？

**A**: 支持。需要：
- Windows 10/11
- PowerShell 或 Git Bash
- Node.js for Windows

### Q4: 如何关闭状态条？

**A**: 直接使用 `openclaw` 命令而不是包装器：
```bash
openclaw  # 原生启动，无状态条
```

### Q5: 可以自定义颜色吗？

**A**: 当前版本使用黑白 ASCII，颜色支持计划在 v0.2.0 中添加。

### Q6: 任务历史会保存吗？

**A**: 不会。状态条只显示当前任务，完成后定格显示，不保存到文件。
