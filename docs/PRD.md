# PRD - Product Requirements Document

## Task Status Indicator for OpenClaw

**版本**: 0.1.0  
**日期**: 2026-03-18  
**作者**: 腾云虾

---

## 1. 问题陈述

OpenClaw 执行工具调用时处于"黑箱"状态，用户无法感知：

- AI 是在思考、执行工具、还是卡住了？
- 当前执行到第几步？
- 任务已完成还是报错？
- 一个步骤运行了多久？

这种不确定性导致用户体验差，尤其是在长耗时任务时。

---

## 2. 目标

提供一个**极简、实时、跨平台**的任务状态可视化方案，让用户对 AI 执行过程有清晰感知。

### 关键指标

| 指标 | 目标值 |
|------|--------|
| 状态感知延迟 | < 1 秒 |
| 视觉干扰 | 最小化（固定底部状态条） |
| 部署复杂度 | 零侵入 OpenClaw 核心 |
| 依赖要求 | 仅 Node.js 内置模块 |

---

## 3. 用户场景

### 场景 1: 网页分析任务

**用户输入**: "分析一下这个网站"

**传统体验**:
```
[输入]
[等待... 不知道在干什么]
[结果突然出现]
```

**期望体验**:
```
[输入]
[🦐] 正在执行: browser.open | 已耗时 2s
[🦐] 正在执行: browser.snapshot | 已耗时 5s | 工具链: browser
[🦐] 正在执行: eval | 已耗时 8s | 工具链: browser → eval
✅ [🦐] eval | 完成 | 耗时 12s | 步骤 3
[结果]
```

### 场景 2: 长时间卡住检测

**用户输入**: "处理这个大数据文件"

**传统体验**:
```
[输入]
[等待... 3分钟过去了... 是卡了还是真的在跑？]
[不确定要不要 Ctrl+C]
```

**期望体验**:
```
[输入]
[🦐] 正在执行: read | 已耗时 30s | 最后活动: 30s 前
[░░░▓░░░░░] 步骤 1 | 最后活动: 30s 前  ← 脉冲动画表示仍在运行
...
[🦐] 正在执行: read | 已耗时 3m 12s | 最后活动: 2s 前
✅ [🦐] read | 完成 | 耗时 3m 15s | 步骤 1
```

---

## 4. 成功标准

- [x] 状态刷新延迟 < 1 秒
- [x] 终端显示不干扰正常输出
- [x] 零侵入 OpenClaw 核心代码
- [x] 单文件可运行，无复杂依赖
- [x] 任务完成/失败有明确反馈
- [x] 支持嵌套工具调用链显示

---

## 5. 非目标

- ❌ 不替代 OpenClaw 官方日志
- ❌ 不提供历史任务查询
- ❌ 不统计性能指标（后续版本可考虑）
- ❌ 不支持 GUI 界面（纯终端方案）

---

## 6. 发布标准

- [x] 代码完成并通过测试
- [x] 安装脚本可一键部署
- [x] 文档完整（README + 使用说明）
- [x] 演示视频/GIF（可选）
