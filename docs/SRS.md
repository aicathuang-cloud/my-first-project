## 1. 功能需求 (Functional Requirements)

### FR-1 实时工具名显示
**优先级**: P0  
**描述**: 状态条必须显示当前正在执行的工具名称  
**验收标准**:
- 工具开始调用时立即更新显示
- 工具名截断至 30 字符以内
- 支持嵌套工具显示（如 `browser.snapshot`）

### FR-2 运行时间显示
**优先级**: P0  
**描述**: 显示任务已运行的累计时间  
**验收标准**:
- 格式自适应：`<1s` / `12s` / `2m5s` / `1h30m`
- 刷新频率 500ms
- 精度：秒级

### FR-3 工具调用链
**优先级**: P1  
**描述**: 显示嵌套工具调用的层级关系  
**验收标准**:
- 链长 ≤3：完整显示 `a → b → c`
- 链长 >3：截断显示 `... → d → e`
- 单工具调用时隐藏工具链区域

### FR-4 步骤计数
**优先级**: P1  
**描述**: 显示当前执行到第几步  
**验收标准**:
- 每检测到一个新工具调用，步骤 +1
- 格式：`步骤 N`（不显示总数，避免打脸）
- 初始状态显示`准备中`

### FR-5 进度条可视化
**优先级**: P1  
**描述**: 用 ASCII 进度条提供视觉反馈  
**验收标准**:
- 多步骤任务：按步骤比例填充 `[████░░░░░░]`
- 单步骤长耗时：切换脉冲动画 `[░░░▓░░░░░]`
- 脉冲阈值：3 秒无步骤变化

### FR-6 任务终态显示
**优先级**: P0  
**描述**: 任务完成或失败时定格显示  
**验收标准**:
- 成功：`✅ [🦐] tool | 完成 | 耗时 X | 步骤 N`
- 失败：`❌ [🦐] tool | 失败 | 耗时 X | 错误类型`
- 定格保留在历史输出中，不清理

### FR-7 终端适配
**优先级**: P0  
**描述**: 支持标准 ANSI 终端  
**验收标准**:
- 使用 `\r` 回车符刷新同一行
- 支持光标上移/清行序列
- 兼容 Linux/Mac/Windows PowerShell

## 2. 非功能需求 (Non-Functional Requirements)

### NFR-1 性能
| 指标 | 要求 |
|------|------|
| 状态刷新延迟 | ≤ 500ms |
| 解析延迟 | ≤ 100ms |
| 内存占用 | ≤ 50MB |
| CPU 占用 | ≤ 5% |

### NFR-2 可靠性
- 包装器崩溃不影响 OpenClaw 主进程
- 状态条异常可安全降级（直接透传输出）
- 支持 Ctrl+C 优雅退出

### NFR-3 可维护性
- 零外部依赖（仅 Node.js 内置模块）
- 代码行数 < 1000 行
- 模块化设计（解析/渲染/控制分离）

### NFR-4 兼容性
- Node.js ≥ 18.0
- 终端宽度 ≥ 80 列
- 支持 UTF-8 编码

## 3. 接口规范

### 3.1 状态对象 (StatusObject)

```typescript
interface StatusObject {
  status: 'pending' | 'running' | 'success' | 'error' | 'idle';
  currentTool: string;
  elapsedMs: number;
  toolChain: string[];
  step: number;
  lastActivityMs: number;
  error?: string;
}
```

### 3.2 解析事件 (ParseEvent)

```typescript
interface ParseEvent {
  type: 'tool_start' | 'tool_end' | 'thinking' | 'output';
  tool?: string;
  status?: 'success' | 'error';
  raw: string;
}
```

## 4. 验收测试用例

### TC-1 基本流程
```
输入: 启动包装器，执行简单工具调用
期望: 看到状态条从 running → success 的完整流程
```

### TC-2 嵌套工具链
```
输入: 执行需要多个工具协作的任务
期望: 工具链正确显示为 a → b → c
```

### TC-3 长耗时检测
```
输入: 执行超过 3 秒的单步骤任务
期望: 3 秒后进度条切换为脉冲动画
```

### TC-4 错误处理
```
输入: 执行一个会失败的工具调用
期望: 状态条显示 ❌ 并保留错误信息
```

### TC-5 并发安全
```
输入: 快速连续发送多个指令
期望: 状态条正确跟踪每个任务，无错乱
```
